LLM Fine-Tuning & Evaluation Framework with Hallucination Detection

Context: NLP · LLM Engineering · University of Oulu, 2025
LoRA RAG Claude Code

A comprehensive hallucination detection and mitigation system for LLM-generated content, achieving a 65% reduction in factual errors. The entire project was built using Claude Code, demonstrating AI-assisted software development at scale.

Hallucination Detection

A multi-dimensional validation framework was built combining fuzzy string matching, semantic similarity scoring, and factual consistency checks. The system cross-references generated content against a retrieval-augmented knowledge base to flag likely hallucinations before they reach the user.

Retrieval-Augmented Generation (RAG)

FAISS (Facebook AI Similarity Search) was used to build a dense vector index over the training corpus. At inference time, relevant context chunks are retrieved and injected into the prompt, grounding the model's output in verified information and significantly reducing fabrication.

Fine-Tuning with LoRA

TinyLlama-1.1B was fine-tuned on a dataset of 500K+ recipes using LoRA (Low-Rank Adaptation), which inserts trainable rank-decomposition matrices into the transformer layers. This allows efficient domain adaptation without updating the full model weights, keeping training feasible on consumer hardware.

Stack: Python · PyTorch · Transformers (HuggingFace) · FAISS · sentence-transformers · LoRA · TinyLlama
