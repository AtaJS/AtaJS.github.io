Face & Hand Gesture Recognition System (ESP8266 + Raspberry Pi)

Context: Embedded Systems · Bachelor's Project · University of Tabriz
Embedded OpenCV TensorFlow

A real-time embedded system combining face recognition and hand gesture detection to issue hardware commands over a local network. The system bridges computer vision on a Raspberry Pi with IoT actuation via an ESP8266 microcontroller.

System Design

The Raspberry Pi 400 captures video from a USB camera and runs the vision pipeline. Recognized faces trigger authentication events, while hand gestures are mapped to commands (e.g., open hand → LED on, closed fist → LED off). Commands are sent to the ESP8266 over HTTP, which drives the GPIO outputs accordingly.

Firmware & Multi-threading

The ESP8266 firmware was written entirely in C++ using the Arduino framework, implementing a lightweight HTTP server to receive and execute commands. On the host side, a multi-threaded Python application separates face recognition, gesture detection, and network communication into concurrent threads to maintain real-time performance.

Gesture Classification

MediaPipe Hands was used for hand landmark detection, feeding 21 keypoint coordinates into a TensorFlow classifier trained on custom gesture data. The classifier distinguishes between several gesture classes with low latency, suitable for real-time embedded deployment.

Stack: C++ (ESP8266 / Arduino) · Python · OpenCV · MediaPipe · TensorFlow · Multi-threading · HTTP
