Intelligent Healthcare Query Routing with LLM Agents

Context: LLM · Healthcare · Routing Systems
LLM Healthcare GPT-4o

An intelligent routing system that directs patient healthcare queries to the appropriate service using three different approaches — from rule-based keyword matching to LLM-powered intent understanding. The system was benchmarked on 45 real healthcare queries across five service categories.

Three Routing Approaches

The project compares a rule-based method against two LLM-based approaches. Rule-based routing uses keyword matching and achieves 95.6% accuracy with instant response times, making it a strong baseline. Google Gemini 2.5 Flash pushes accuracy to 97.8% at ~0.877s latency. Azure OpenAI GPT-4o reaches 100% accuracy at 0.466s — faster and more accurate than Gemini, though at a higher cost.

Why LLMs Win on Ambiguous Queries

The key advantage of LLM routing is context understanding. A rule-based system struggles to distinguish between "schedule an appointment" and "cancel an appointment" if the keyword "appointment" triggers the same category. LLMs correctly resolve intent even in ambiguous or multi-part queries, which is critical in a healthcare setting where misrouting can cause real harm.

Resilience & Cost

The system includes graceful fallback to rule-based routing during API outages, ensuring 100% uptime. A cost-benefit analysis showed GPT-4o costs approximately $0.50 per 1,000 queries — justified for high-stakes clinical routing where accuracy matters most.
